    STF之Provider系列六：brew

    2020-03-07
    jiansh / STF
    jiansh

    link on JianShu

    STF之Provider系列五支持脚本自动部署。目前国内到网络环境导致使用brew install 各种依赖应用时：1）耗时；2）不稳定。

    STF之Docker for Mac失败得出结论，Mac环境下到Docker无法挂载USB设备。

    需要利用brew的本地化安装来提供环境部署——

    How to install a local file in Homebrew
    How to use Homebrew to install local archive

    # download your archive
wget https://github.com/zeromq/libzmq/releases/download/v4.3.2/zeromq-4.3.2.tar.gz

# move the archive to your brew --cache
mv zeromq-4.3.2.tar.gz $(brew --cache)

# then install the archive as normal
brew install zeromq

    使用brew cat zeromq查看zeromq的下载地址

    掌握brew的基础知识，使用手册，常见问题

    基本思路是将STF依赖的各个包graphicsmagick zeromq protobuf yasm pkg-config，以及每个包本身的依赖都下载到本地，然后利用本地化安装进行安装。从而降低对网络对依赖

    使用brew deps -n Formula 可以查看对应包到依赖，通过 brew cat foumular可以查看到可下载到url地址是什么。

    gist for install protobuf

    $ wget https://github.com/google/protobuf/releases/download/v3.4.1/protobuf-cpp-3.4.1.tar.gz
$ tar -zxvf protobuf-cpp-3.4.1.tar.gz
$ sudo mv protobuf-3.4.1 /usr/local/bin
$ cd /usr/local/bin/protobuf-3.4.1
$ ./configure
$ make 
$ make install
$ protoc --version
